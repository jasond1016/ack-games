# Separate Racing Track Shape From Race Mode

The racing map system supports two explicit track shapes, `open` and `loop`, and keeps race mode as start-of-race configuration rather than a map-intrinsic concept. For now, compatibility is intentionally one-to-one: `open` tracks run point-to-point sprint races and `loop` tracks run lap races, because the two shapes need different finish semantics, progress rules, and editor affordances. Open tracks derive direction from control-point order, auto-generate start and finish lines, keep the finish line slightly inset to leave a finish buffer, and remove standalone obstacle editing in favor of automatically generated side rails; loop tracks keep an editable start position on the track and finish immediately on the winning lap, while both shapes share strict pre-commit validation for self-intersections, minimum control-point spacing, and full-map acceptance on import.
